Git — Command Arsenal (left) + Practice Tasks (right)

Command Arsenal

Check installation
git --version
Verify Git is installed.
Set identity & view config
git config --global user.name  "Your Name"
git config --global user.email "you@example.com"
git config --global -l
Configure author name/email for commits; list global settings.
Create workspace & init
mkdir my-git-project
cd my-git-project
git init
ls -a
Make a folder, initialize a repo, and confirm the .git/ directory exists.
Status → stage → commit
touch somefile.txt
git status
git add somefile.txt    # or: git add .
git status -s
git commit -m "Add initial file"
Create a file; inspect status; stage changes; commit with a clear, imperative subject.
History (basic & compact)
git log
git log --oneline --graph
View commit history; the compact graph version is great on screen.
Branches & merge
git branch
git branch my-new-branch
git switch my-new-branch
git switch main
git merge my-new-branch
List/create/switch branches and merge back to main.

Practice Tasks

  1. Confirm Git is installed and accessible from the terminal.
  2. Set your author identity (name + email) globally. Then view your global config to confirm.
  3. Create a workspace: make a new folder, enter it, and initialize a Git repository.
  4. Create your first commit: add a file, stage it, and write a clear message in the imperative mood (as if giving an instruction). Pick one verb:
    AddFixUpdateRemove RefactorRenameDocumentTest RevertBumpConfigureBuild CIChore
    Example (subject only): Add initial file or Fix remove invalid path.
  5. Create a new branch and switch to it. On that branch, create a brand‑new file (e.g., feature.txt), stage, and commit (imperative subject).
  6. Return to main and inspect the history with the compact graph. Do you see two tips diverging?
  7. Merge your branch back to main. Verify that the new file from the branch now exists on main too, and the history shows the merge point.

Desired outcome (in words)

Two branches — main and my-new-branch. The branch introduces a new file. After merging, that file is present on main as well.
