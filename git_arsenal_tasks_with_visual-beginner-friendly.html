Git + GitHub — Command Arsenal (left) + Two-Branches Lab (right)

Command Arsenal (GitHub-centric)

Connect local repo to GitHub
git remote add origin <REMOTE-URL>
git remote -v
Use the SSH/HTTPS URL of your GitHub repository.
Rename master → main & first push
git branch -M main
git push -u origin main
Sets main as the primary branch and establishes upstream.
Sync before branching
git switch main
git pull origin main
Create & publish a branch (first time)
git switch -c <branch>
echo "Changed by <branch>" >> file.txt
git add .
git commit -m "Add line from <branch>"
git push -u origin <branch>
-u sets tracking so next time a simple git push/git pull works.
Open & fix PR conflict locally (on PR branch)
git switch <pr-branch>
git fetch origin
git merge origin/main
# edit files → resolve
git add <files>
git commit
git push
Push after resolving to update the PR on GitHub.

Two-Branches Lab (GitHub)

  1. Create an empty repo on GitHub (no README/License).
  2. Connect local to GitHub with git remote add origin <REMOTE-URL> and verify with git remote -v.
  3. Rename to main and push: git branch -M main → git push -u origin main. Confirm files appear in the portal.
  4. Create branch-a: sync main, then create the branch, append a line, stage and commit (imperative), and publish with tracking: 
    git switch main
git pull origin main
git switch -c branch-a
echo "Changed by branch-a" >> file.txt
git add .
git commit -m "Add line from branch-a"
git push -u origin branch-a
  5. Create branch-b: sync main again, create branch, change, commit, and publish with tracking: 
    git switch main
git pull origin main
git switch -c branch-b
echo "Changed by branch-b" >> file.txt
git add .
git commit -m "Add line from branch-b"
git push -u origin branch-b
  6. Open PRs in GitHub: branch-a → main and branch-b → main. If both touched the same line, the second PR will show a conflict.
  7. Resolve conflict locally on the PR branch (recommended): 
    git switch branch-b
git fetch origin
git merge origin/main     # conflict appears if A already merged
# edit file(s), remove markers, keep/merge content
git add file.txt
git commit
git push
    GitHub will update the PR and allow merging.

Desired outcome (in words)

Two branches — branch-a and branch-b — both published to GitHub. Each has its own change; after resolving any conflict, both changes merge into main.
