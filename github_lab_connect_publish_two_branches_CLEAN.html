GitHub Lab — Connect & Publish (Clean)

Command Arsenal (GitHub)

Connect local repo to GitHub
git remote add origin <REMOTE-URL>
git remote -v
Use your repo URL (SSH or HTTPS) from GitHub.
Rename master → main & first push
git branch -M main
git push -u origin main
Sets main as primary and establishes upstream.
Create & publish branch (first time)
git switch -c <branch>
echo "Changed by <branch>" >> file.txt
git add .
git commit -m "Add line from <branch>"
git push -u origin <branch>
Sync with main before branching
git switch main
git pull origin main
Resolve PR conflict locally (on PR branch)
git switch <pr-branch>
git fetch origin
git merge origin/main
# edit files → resolve
git add <files>
git commit
git push
Push after resolving to update the PR.

Scenario — Step by Step

1) Create empty repo on GitHub

Log in and create a new repository (leave README/License unchecked).

2) Connect local repo to GitHub

git remote add origin <REMOTE-URL>
git remote -v

3) Rename primary to main (if needed) & push

git branch -M main
git push -u origin main

Open the repo page in GitHub and confirm files are visible.

4) Prepare two branches

git switch main
git pull origin main
git switch -c branch-a
echo "Changed by branch-a" >> file.txt
git add .
git commit -m "Add line from branch-a"
git push -u origin branch-a
git switch main
git pull origin main
git switch -c branch-b
echo "Changed by branch-b" >> file.txt
git add .
git commit -m "Add line from branch-b"
git push -u origin branch-b

5) Open PRs in GitHub

Create PRs: branch-a → main and branch-b → main. If both changed the same line, the second PR will show a conflict.

6) Resolve the conflict locally (on PR branch)

git switch branch-b
git fetch origin
git merge origin/main     # conflict happens here if A was merged first
# edit files to resolve (remove <<<<<<<, =======, >>>>>>> markers)
git add file.txt
git commit
git push

GitHub will update the PR and allow merging.

Tip: if your default branch is master, replace main with master in all commands.
